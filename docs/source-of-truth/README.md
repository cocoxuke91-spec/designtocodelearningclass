# Pmagic AI landing page — source of truth

This directory is the durable reference for reconstructing the supplied landing-page screenshot as a responsive, production-grade HTML website.

## Evidence hierarchy

When sources conflict, use this order:

1. `reference/agent-core-primary.png` — primary visual evidence for the central 3D agent-core artwork only.
2. `reference/pmagic-ai-original.png` — primary visual evidence for the full page, including layout, labels and surrounding composition.
3. `source-manifest.json` — machine-readable identity, scope and provenance of all primary evidence and derived assets.
4. `reference/slices/` and `slice-manifest.json` — reproducible derived evidence for section-level work.
5. `visual-spec.md` — observed and derived design facts.
6. `acceptance-criteria.md` — measurable completion requirements.
7. `implementation-plan.md` — implementation approach and sequencing.
8. `decision-log.md` — explicit engineering decisions and assumptions.

Authority is scoped: `agent-core-primary.png` wins only for the central artwork's geometry, materials, colors, proportions and camera angle. The full-page screenshot remains authoritative for placement, scale relative to copy, agent labels and overall page composition.

## Evidence labels

- **Observed**: directly visible in the supplied image.
- **Measured**: obtained from the source file or tooling.
- **Derived**: inferred from proportions or common browser capture behavior.
- **Decision**: deliberately chosen for implementation; not claimed to be present in the source.
- **Unknown**: cannot be established from the image alone.

Do not silently convert a derived value or decision into an observed fact. When implementation uncovers a mismatch, update the relevant document and add an entry to the decision log.

## Reference slices

Section slices are **derived facts**, not independent primary sources. They are lossless crops of the primary screenshot and exist to make section-level implementation and visual regression easier. Their crop rectangles, parent source and hashes are recorded in `slice-manifest.json`; they can be regenerated by running `scripts/create-reference-slices.sh`.

If a slice conflicts with the full screenshot, the full screenshot wins. Do not edit a slice by hand.

## Primary limitation

The original artifacts are flattened raster images. They do not contain the original fonts, vector icons, 3D models, responsive rules, DOM semantics, animation timing, interaction states, or exact copy outside what is legible in the raster images. Those items must be reconstructed and documented as decisions.
